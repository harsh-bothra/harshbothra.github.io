1. IP Addressing

1.1 Classful Addressing (IPv4)

Legacy IPv4 divided the 32-bit address space into five classes (A–E), each with a fixed subnet mask and number of usable host addresses.

C192.0.0.0–223.255.255.255/24 (255.255.255.0)254
D224.0.0.0–239.255.255.255MulticastN/A

1.2 CIDR (Classless Inter-Domain Routing)

CIDR allows variable-length subnet masks. Notation: IP/prefix. Flexibly allocate addresses and reduce waste.

CIDRMaskTotalUsable
/8255.0.0.016,777,21616,777,214

Example: 192.168.1.0/26 creates subnets 192.168.1.0–.63, .64–.127, etc.

2. MAC Address

Media Access Control addresses are 48-bit identifiers (e.g., 00:1A:2B:3C:4D:5E). First 24 bits (OUI) denote vendor, last 24 bits NIC-specific. Switches build MAC tables to forward frames. ARP resolves IP to MAC within LAN.

3. Process Identifier (PID)

PIDs uniquely identify running OS processes. Commands like netstat -tupn or lsof -i map PIDs to network sockets, crucial for isolating suspicious listeners during incident response.

4. OSINT Model

The OSINT lifecycle: Collection (WHOIS, social media), Processing (filter noise), Analysis (correlate events), Dissemination (report findings). Tools include Maltego, Shodan, theHarvester.

UDP is connectionless and lightweight—suitable for DNS, VoIP, streaming—but lacks built-in reliability.

7. TCP Three-Way Handshake

SYN: Client→Server (seq=X).
SYN-ACK: Server→Client (seq=Y, ack=X+1).
ACK: Client→Server (ack=Y+1).

This exchange establishes state and initial sequence numbers. Monitoring for incomplete handshakes detects SYN flood attacks.

8. Packet Encapsulation

Data is wrapped per OSI layers:

LayerHeader
Data LinkMAC src/dst, EtherType
NetworkIP src/dst, TTL, Protocol
TransportPorts, Seq/Ack or Length, Flags
ApplicationProtocol payload (HTTP, DNS, etc.)

9. VPN & Tunneling

VPNs secure traffic over public networks via tunnels. Common protocols:

  • IPsec: IKE for key exchange (UDP 500), ESP for payload encryption, NAT-T (UDP 4500).
  • OpenVPN: SSL/TLS-based, custom ports.
  • WireGuard: Modern, minimal code, uses UDP and public-key crypto.

10. DNS & Resolution

DNS resolves hostnames to IPs. Two query types:

  • Recursive: Resolver asks root, TLD, and authoritative servers.
  • Iterative: Client queries each server in turn.

DNSsec adds signatures (RRSIG) and DS records to ensure authenticity and integrity.

11. ARP & MAC Resolution

ARP maps IP to MAC within a LAN. An ARP request ("Who has IP?") is broadcast; the owner replies. ARP spoofing poisons caches, enabling man-in-the-middle.

12. DHCP Workflow

DHCP lease process: DISCOVER (client broadcast), OFFER (server reply), REQUEST (client selects), ACK (server confirms). Configuration includes IP, mask, gateway, DNS.

13. DoS & DDoS

DoS: Single source floods target (e.g., SYN flood).
DDoS: Distributed via botnet amplifies volume (DNS/ NTP).

Mitigation: rate limiting, blackholing, upstream scrubbing services.

14. Intrusion Detection Systems (IDS)

NIDS: Monitors network packets.
HIDS: Monitors host logs and processes.
Signature-based detects known patterns; anomaly-based finds deviations from baseline.

15. Firewalls

Types:

  • Packet-filter: ACLs on IPs/ports.
  • Stateful: tracks connection states.
  • NFGW: inspects application layer, user identity.

16. Intrusion Prevention Systems (IPS)

Inline systems that block detected threats (e.g., resets TCP connections). Balance sensitivity to avoid false positives.

17. Honeypots & Honeynets

Decoy systems to attract attackers. Low-interaction emulate services; high-interaction run real OS. Honeynets network multiple honeypots for deeper threat analysis.

18. Security Operations Center (SOC)

Central team monitors security incidents. Layers:

  • L1: Triage alerts.
  • L2: Investigation.
  • L3: Threat hunting and forensics.
