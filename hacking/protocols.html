Networking - Protocols

Key Protocols Explained

HTTP (Port 80/TCP)

Hypertext Transfer Protocol powers unencrypted web traffic. It follows a request-response model, with clients sending GET or POST requests to servers hosting websites. HTTP is foundational for web browsing but lacks confidentiality unless secured.

HTTPS (Port 443/TCP)

HTTP over TLS/SSL ensures encrypted communication between client and server. HTTPS protects data integrity and privacy by wrapping HTTP within a secure TLS layer, preventing eavesdropping and tampering.

FTP (Ports 20,21/TCP)

File Transfer Protocol enables file uploads and downloads via separate control (21) and data (20) channels. Standard FTP transmits credentials unencrypted, making it vulnerable to interception unless secured by FTP over TLS (FTPS).

SSH (Port 22/TCP)

Secure Shell provides encrypted remote shell access and secure file transfers (SCP, SFTP). SSH is a cornerstone for secure administration, supporting key-based authentication and tunneling.

SMTP (Ports 25,587,465/TCP)

Simple Mail Transfer Protocol handles email sending. Port 25 is for server-to-server relays, 587 for client submissions (with STARTTLS), and 465 for SMTP over implicit TLS. Misconfigured SMTP can lead to spam relay or credential leaks.

IMAP (Ports 143,993/TCP)

Internet Message Access Protocol allows email retrieval and synchronization across devices. Unsecured on port 143, but encrypted on 993 via TLS. IMAP supports complex mailbox operations, making it a frequent target for credential harvesting.

POP3 (Ports 110,995/TCP)

Post Office Protocol v3 downloads emails to a single client, optionally deleting from the server. Unencrypted on 110 and secure on 995 (POP3S). Less favored today due to limited server-side management.

DNS (Port 53/UDP, TCP)

Domain Name System translates hostnames to IP addresses. Most queries use UDP, while zone transfers and large responses use TCP. DNS poisoning and cache spoofing remain critical attack vectors.

DHCP (Ports 67,68/UDP)

Dynamic Host Configuration Protocol assigns IP addresses and network settings. Clients broadcast on 68, servers listen on 67. Rogue DHCP servers can mislead clients to malicious gateways.

TFTP (Port 69/UDP)

Trivial File Transfer Protocol is a lightweight FTP variant without authentication, often used for network booting and firmware updates. Its lack of security makes it an easy vector for unauthorized file transfer.

SNMP (Ports 161,162/UDP)

Simple Network Management Protocol monitors and configures network devices. SNMPv1/v2 use community strings as weak 'passwords.' SNMPv3 adds encryption and robust authentication.

NTP (Port 123/UDP)

Network Time Protocol synchronizes system clocks. Unsafeguarded NTP servers can be abused for reflection DDoS attacks or time-based authentication bypass.

Telnet (Port 23/TCP)

Telnet offers unencrypted remote shell access. Largely deprecated in favor of SSH, but still found on legacy systems. Captured Telnet credentials are in clear text.

LDAP (Ports 389/UDP, 636/TCP)

Lightweight Directory Access Protocol structures directory services, like Active Directory. Plain LDAP listens on 389; LDAPS on 636 uses TLS. Misconfigured LDAP can leak user and group information.

SMB/CIFS (Port 445/TCP)

Server Message Block facilitates Windows file and printer sharing. Vulnerable services can be exploited via EternalBlue or credential theft through NTLM relay attacks.

RDP (Port 3389/TCP, UDP)

Remote Desktop Protocol allows Windows GUI remote access. Exposed RDP endpoints are high-value targets for brute-force, BlueKeep, and other RDP-specific exploits.

MQTT (Ports 1883,8883/TCP)

Message Queuing Telemetry Transport is a publish/subscribe protocol for IoT. 1883 is plaintext, while 8883 uses TLS. Unauthorized access can manipulate IoT devices.

SIP (Ports 5060/UDP, 5061/TCP)

Session Initiation Protocol establishes VoIP sessions. Port 5060 for unencrypted, 5061 for TLS. SIP misconfigurations can enable eavesdropping or toll fraud.

IPsec (Ports 500,4500/UDP; ESP, AH/IP)

IPsec secures IP communications via ESP and AH. UDP ports 500 and 4500 support key exchange (IKE). Properly configured IPsec tunnels protect networks but can be bypassed if mismanaged.

GRE (Protocol 47/IP)

Generic Routing Encapsulation tunnels various network layer protocols. Used for VPNs but lacks native encryption, requiring pairing with IPsec for confidentiality.

ICMP (Protocol 1/IP)

Internet Control Message Protocol sends diagnostic (echo request/reply) and error messages. ICMP can be abused for network mapping (ping sweeps) or reflection attacks.
